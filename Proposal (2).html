pACID-Bench: A Parametric Complexity Analysis of Probabilistic Knowledge Contracts for Large Language Models

Deriving tight achievability bounds for pACID(α, ε, δ) parameters across model architectures, capacity regimes, and knowledge distributions — converting aspirational contracts into provable feasibility certificates.

pACID formalization probabilistic guarantees parametric complexity failure curve analysis learning theory NeurIPS VLDB
2 · Abstract
The pACID framework, introduced in the PDB position paper, specifies probabilistic analogues of ACID for parametric knowledge systems: α-atomicity, ε-isolation, and δ-semantic consistency. However, the current formulation is definitional, not operational: it states what guarantees should mean but not which (α, ε, δ) triples are achievable for a given model architecture, capacity, and knowledge domain. We introduce pACID-Bench, a parametric benchmark that empirically and theoretically characterizes the pACID feasibility space. We decompose pACID compliance into six orthogonal complexity dimensions — capacity ratio, knowledge density, semantic entanglement, update frequency, query distribution shift, and adversarial pressure — and vary each independently to produce failure curves rather than scalar metrics. Our key result is the pACID Feasibility Map: a characterization of the achievable contract space as a function of model architecture, showing precisely where current LLMs fall and what architectural changes would expand the achievable region. We prove tight information-theoretic lower bounds on minimum achievable δ (semantic consistency gap) as a function of training set coverage, and empirically validate the Isolation–Generalization Tradeoff formula across 12 model families.
3 · Motivation & Research Gap

The Single-Score Problem in PDB Evaluation

Existing LLM evaluation benchmarks (MMLU, TruthfulQA, NaturalQuestions) report scalar accuracy scores that conflate multiple failure modes. For a Parametric Database, this is catastrophically insufficient: a model with 80% accuracy may satisfy α-atomicity trivially while completely violating δ-consistency on adversarial queries, or it may have excellent ε-isolation but catastrophic hallucination on rare knowledge. Single-score metrics cannot distinguish these failure profiles.

Lack of Controlled Experimentation on pACID Parameters

No existing work parametrically varies the conditions that affect pACID compliance — knowledge density, capacity ratio, semantic entanglement between target and collateral facts — while holding all other factors fixed. Without such controlled experiments, it is impossible to diagnose why a system fails a particular pACID property or predict its failure threshold as conditions change.

Absence of Parametric Scaling Laws for Knowledge Contracts

Scaling laws for next-token prediction loss are well-established (Chinchilla). Analogous laws for pACID compliance — how α, ε, δ scale with model size, training compute, and knowledge density — do not exist. Without them, engineers cannot reason about how much capacity is needed to achieve a given contract specification.

5 · Generation & Verification Pipeline
01
Parametric Fact Generator: Generates synthetic knowledge bases with controlled density, entanglement, and semantic clustering using a knowledge graph skeleton (Wikidata subgraph). Each instance is fully specified: target fact k, collateral set {k'}, semantic neighborhood radius r.
02
Complexity Level Controller: Independently varies each of D1–D8 while holding others fixed. Generates difficulty levels 1→10 per dimension using deterministic perturbation functions (e.g., Gaussian entanglement injection, controlled synonym expansion for D3).
03
pACID Verifier: Measures α (PIF score), ε (blast radius Β), δ (hallucination rate H) for each generated instance using automated querying with ground-truth labels. No human annotation required.
04
Data Filter: Removes degenerate instances (trivially trivial: PIF=1.0 at level 1; trivially impossible: PIF=0 at all levels) using a two-pass filter. Retains only instances with discriminating failure curves.
05
Natural Language Renderer: Converts parametric instances into natural language query sets via templated cloze completion, free-form QA, and adversarial rephrasing. Contamination-proof: all test queries are generated post-training cutoff using a held-out template set.
06
Evaluation Protocol: Models are evaluated per dimension, per level. Output is an 8D failure curve surface rather than a scalar. Aggregation produces pACID coordinates (α̂, ε̂, δ̂) for each model.
6 · Experimental Design

Models: 7B–70B dense transformers (Llama-3, Mistral), MoE models (Mixtral), memory-augmented (RETRO), edited models (ROME/MEMIT baselines), and LoRA-adapted variants. Agent systems evaluated separately with tool augmentation.

Metrics (curves not scalars): For each dimension Di at level ℓ, report: PIF(ℓ), Blast Radius Β(ℓ), Hallucination Rate H(ℓ), Adversarial Recovery Probability ARP(ℓ). Plot failure curves: metric vs. level. Report threshold levels (e.g., level at which PIF drops below 0.90).

Evaluation Setup: 1000 instances per dimension per level; 3 random seeds. Statistical significance via paired bootstrap. Report mean ± 95% CI failure curves.

7 · Hypotheses
H1 (Capacity Threshold) There exists a critical capacity ratio d*/n below which α-atomicity cannot exceed 0.85 for any gradient-based editing method, independent of the editing algorithm used.
H2 (Entanglement–Isolation Tradeoff) ε-isolation degrades monotonically with semantic entanglement (D2), and the degradation curve follows a power law: Β(entanglement) ∝ entanglement^γ for γ > 1, independent of model size.
H3 (Agent vs. Base Model) Agent systems with external memory achieve significantly higher α-atomicity (>+0.15) but lower ε-isolation (-0.10) than base parametric models, due to external storage enabling atomic commits but increasing cross-module leakage.
H4 (Adversarial Pressure Cliff) δ-consistency exhibits a phase transition rather than smooth degradation as adversarial budget (D6) increases — a cliff at budget level k* where consistency drops >40% in one step.
H5 (Scaling Law for δ) Minimum achievable hallucination rate δ scales as δ_min ∝ n^{-β} · coverage^{-γ}, where n is model scale, coverage is training corpus coverage of the knowledge domain, and β, γ are universal constants independent of model family.

9 · Application: Cost Routing

The pACID Feasibility Map enables a Contract-Aware Query Router: given an incoming query with declared pACID requirements (α ≥ 0.95, δ ≤ 0.02), the router matches the query's estimated complexity profile (via D1–D8 dimension classifiers) against precomputed pACID coordinates of available models, routing to the cheapest model whose feasibility map covers the required contract. Estimated cost savings of 30–50% by avoiding frontier model calls for queries where smaller models satisfy the required contract.

ErasureProof: A Parametric Complexity Analysis of Certified Deletion in Overparameterized LLMs

Closing the suppression–erasure gap through DP-certified parametric deletion, adversarially parameterized difficulty scaling, and tight lower bounds on minimum achievable Adversarial Recovery Probability.

machine unlearning differential privacy certified deletion GDPR compliance influence functions ICML CCS
2 · Abstract
Machine unlearning for LLMs faces a fundamental gap: current gradient-based suppression methods achieve behavioral suppression under standard prompting but fail certified erasure against adversarial recovery. Sinha et al.'s Step-by-Step Reasoning Attack demonstrates Adversarial Recovery Probabilities of 0.60–0.80 after standard unlearning — far above GDPR-compliant thresholds. We introduce ErasureProof, a parametric benchmark and algorithm framework for certified parametric deletion. We decompose the deletion problem into 7 orthogonal complexity dimensions — knowledge recoverability, semantic entanglement depth, adversarial budget, model capacity, deletion set size, reasoning chain length, and training frequency of the target fact — and produce failure curves across each. Our core algorithmic contribution is Hessian-Free Certified Deletion (HFCD): an extension of the Guo et al. DP-certified deletion framework to overparameterized transformers using Kronecker-factored curvature approximations, achieving (ε, δ)-certification on facts with >100 training appearances. We prove that ARP ≥ f(training frequency, model capacity) for any gradient-based deletion method, establishing a fundamental lower bound on the suppression–erasure gap.
3 · Motivation & Research Gap

The Suppression–Erasure Gap: Gradient ascent unlearning and related methods produce models that output refusals under standard prompting but leak the "deleted" knowledge through structured multi-step reasoning. This is not a marginal failure — it is a systematic one that the Step-by-Step Reasoning Attack exploits with >60% success. The gap exists because gradient-based suppression changes the output distribution over standard prompts but does not change the geometry of knowledge representations in weight space.

The Certification Void: No current LLM unlearning method provides machine-checkable, adversary-agnostic certification. SISA training provides strong guarantees but requires the original training data and is computationally prohibitive. Influence function methods provide (ε, δ)-certification only for strongly convex models — inapplicable to transformers. The field lacks a tractable certification framework for the overparameterized regime.

The Benchmark Gap: Existing unlearning evaluations (TOFU, MUSE) use fixed attack budgets and fixed knowledge types. No benchmark parametrically scales adversarial pressure, knowledge recoverability, and reasoning chain length independently, making it impossible to characterize where deletion methods fail and why.

PROPOSAL P03 · THEME B — PARAMETRIC CRUD

CalibRead: Parametric Complexity Analysis of Probabilistic Read Reliability in LLM Knowledge Systems

Replacing scalar hallucination rates with calibrated failure curves across query type, knowledge density, and uncertainty — enabling PDB Read operations with formally bounded false-positive retrieval rates.

hallucination detection calibration conformal prediction selective prediction parametric read ICLR EMNLP
2 · Abstract
The Read operation of a Parametric Database must provide calibrated confidence alongside generated answers — a (y, c) pair where c bounds the probability of y being factually incorrect. Current systems return y with no formal confidence guarantee: calibration research shows that LLM confidence is systematically miscalibrated across knowledge types, model sizes, and query distributions. We introduce CalibRead, a parametric benchmark that characterizes LLM Read reliability as a function of 7 independently controlled complexity dimensions, producing hallucination rate curves rather than scalar metrics. Our core technical contribution is Conformal Parametric Read (CPR): an algorithm that wraps any LLM's generation with conformal prediction guarantees, producing prediction sets with provable coverage: Pr(correct answer ∈ prediction set) ≥ 1−α for any α, with minimal set size. We demonstrate that hallucination rate H(θ, Q) is not a fixed property of a model but a well-structured function of query complexity, enabling precise domain-specific reliability contracts.

PDB-TPC: A Parametric Database Benchmark Suite for Evaluating Knowledge Management Operations

The TPC-C/TPC-H analogue for Parametric Databases — standardized, contamination-proof benchmarks for all four CRUD operations, covering atomicity, interference, idempotency, deletion verification, and semantic consistency at scale.

benchmarking systems evaluation CRUD operations parametric database community standard VLDB NeurIPS D&B
2 · Abstract
The field of Parametric Databases lacks a standardized evaluation framework. Research on model editing, continual learning, machine unlearning, and hallucination detection proceeds with incompatible metrics, different knowledge domains, and unstandardized experimental protocols — making cross-paper comparison nearly impossible. We introduce PDB-TPC, a community-standard benchmark suite for Parametric Database evaluation, structured as four independent modules analogous to TPC workloads: PDB-C (Create/atomicity, interference, semantic consistency), PDB-R (Read/hallucination, calibration, idempotency), PDB-U (Update/blast radius, rollback fidelity, isolation), and PDB-D (Delete/suppression-erasure gap, ARP, cascade control). Each module parametrically varies 5–7 complexity dimensions, generating infinite scalable instances. All instances are contamination-proof (generated post-training-cutoff via templated synthesis). PDB-TPC is the first unified framework that enables direct comparison of architecturally diverse systems (dense LLMs, adapter-based PDBs, hybrid RAG systems) on a common operational contract.
3 · Motivation & Research Gap

The Incompatibility Problem: Model editing papers evaluate on CounterFact; unlearning papers evaluate on TOFU/MUSE; continual learning papers evaluate on TRACE/LLMBar. These benchmarks use different knowledge domains, different metrics, and different experimental protocols. A unified benchmark is a prerequisite for the PDB paradigm to function as an engineering discipline rather than a collection of techniques.

Missing Operations: No existing benchmark evaluates all four CRUD operations jointly. Particularly absent: standardized Update-with-rollback evaluation, joint atomicity-and-interference measurement for Create, and cross-operation interference (does a Delete degrade Read quality on related facts?).

The Contamination Crisis: As LLMs are trained on increasingly comprehensive web data, benchmarks are increasingly contaminated — models may have memorized benchmark answers during pretraining. PDB-TPC addresses this through full synthetic generation post-cutoff, enabling perpetually fresh evaluation.

CascadePDB: Parametric Complexity Analysis of Semantic Cascade Specification and Execution in Knowledge Deletion

Formalizing how parametric Delete operations should propagate through knowledge dependency graphs — defining a cascade specification language, analyzing execution correctness across complexity dimensions, and proving bounds on achievable cascade precision.

2 · Abstract
Classical database Delete operations with cascade rules specify precisely which dependent records should be deleted (CASCADE) or nullified (SET NULL) when a referenced record is removed. Parametric Delete has no cascade specification mechanism: deleting one fact may suppress directly derived facts, leave logically dependent facts intact, or propagate unpredictably through the knowledge graph. We introduce CascadePDB, a framework for semantic cascade specification in parametric deletion. We develop: (1) a formal cascade specification language (CSL) for declaring intended propagation behavior — "delete k and all facts directly derivable from k but not facts that merely share an entity with k," (2) a cascade execution algorithm that maps CSL specifications to parametric unlearning procedures, and (3) a parametric benchmark characterizing cascade precision and recall across 7 complexity dimensions. Our key result: cascade execution is achievable at ≥85% precision for first-order derivability cascades but degrades to ≤50% for multi-hop derivability, establishing a practical boundary for automated cascade specification.

CapBound: Information-Theoretic Capacity Bounds for Parametric Knowledge Systems — Deriving Tight Limits on Achievable pACID Contracts

Grounding the Isolation–Generalization Tradeoff in information-theoretic capacity arguments — proving tight bounds on maximum achievable PIF, minimum achievable blast radius B, and the fundamental knowledge density limit for any finite-parameter system.

information theory capacity bounds isolation-generalization tradeoff knowledge density Shannon capacity NeurIPS Theory ICML
2 · Abstract
The PDB position paper establishes the Isolation–Generalization Tradeoff as a formal proposition: the product of isolation strength and generalization capability is bounded by representational capacity d/n. However, this bound is derived from a stylized capacity argument and requires tightening: are there architectures that achieve the bound with equality? What is the exact shape of the tradeoff surface? How does the bound change for structured knowledge distributions (e.g., knowledge graphs with power-law degree distributions)? We introduce CapBound, a theoretical framework that derives tight information-theoretic capacity bounds for parametric knowledge systems using mutual information, rate-distortion theory, and the information bottleneck method. Our central contribution is the Parametric Knowledge Capacity Theorem: the maximum number of facts n* encodable in a d-parameter model at pACID level (α, ε, δ) is exactly n* = d · I(θ; K) / R(α, ε, δ), where I(θ; K) is the mutual information between parameters and knowledge, and R(α, ε, δ) is a rate function derived from the pACID contract. This converts the tradeoff from an inequality to an equality, enabling capacity planning for PDB deployments.

Step 5 · Self-Refinement & Final Reflection

After collective review, the 15 proposals were assessed for redundancy, divergence, and novelty coverage. Three refinements were applied: (1) P12 (ParaMVCC) was strengthened relative to an earlier version that overlapped with P06 by emphasizing the concurrency and conflict resolution aspects rather than the adapter architecture. (2) P15 (CapBound) was refocused from general scaling laws to tight pACID-specific bounds to avoid overlap with P01. (3) P10 (RegPDB) was given an interdisciplinary frame and hybrid-architecture focus to ensure it is not reducible to P02 (ErasureProof).

Coverage of Domains

The 15 proposals span: formal theory (P01, P11, P15), empirical systems (P03, P05, P06, P08, P12), security/compliance (P02, P07, P10), evaluation/standards (P04, P13, P14), and systems architecture (P06, P09). No domain is represented more than 4 times. Cross-cutting problems (P01 touches theory + empirical; P04 touches systems + evaluation) provide thematic cohesion.

Diversity of Approaches

Approaches represented: information-theoretic proofs (P01, P15), impossibility theorems (P11), differential privacy algorithms (P02), conformal prediction (P03), systems benchmarking (P04), transaction protocols (P05, P09), adapter architecture (P06, P12), causal tracing (P08), adversarial evaluation (P07), legal analysis (P10), neurosymbolic verification (P13), formal language design (P14). No two proposals share a primary methodology.

Publication Potential

Strong NeurIPS/ICML potential: P01, P02, P11, P15 (theory + algorithms). Strong VLDB potential: P04, P06, P12 (systems). Strong ICLR potential: P03, P05, P08, P09 (ML methods). Strong CCS/security potential: P07, P10 (security + compliance). Strong EMNLP potential: P13, P14 (NLP + semantics). All 15 proposals have a clearly identified primary venue and would be competitive at the identified targets.

Lab Direction Coherence

Each of the 5 themes forms a coherent 2–3 year research agenda. Theme A (Theory) produces foundational results that underpin all other work. Theme B (CRUD) provides the core algorithmic contributions. Theme C (Compliance) addresses the most commercially pressing problem. Theme D (Architecture) delivers deployable infrastructure. Theme E (Evaluation) provides community infrastructure and enables fair comparison across research groups.

Dependency Structure

Proposed execution order: Theory (P01, P11, P15) → Benchmarks (P04) → CRUD algorithms (P02, P03, P05, P08) → Architecture (P06, P09, P12) → Compliance (P07, P10) → Semantics (P13, P14). Most proposals have dependencies on P04 (benchmark) and benefit from P01 (contract theory), making those two the highest-priority initial deliverables.

P11 (ImpossDB) has the lowest feasibility score due to the mathematical depth required; it may benefit from collaboration with a formal methods group. P10 (RegPDB) requires genuine legal expertise alongside ML — an interdisciplinary challenge. P15 (CapBound) requires reliable I(θ; K) estimation, which is itself an open problem; an early sub-paper on estimation methods may be necessary before the main theorem paper.
