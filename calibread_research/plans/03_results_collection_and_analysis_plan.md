# Results collection and analysis plan

## 1. Principle

Raw generations, parsed answers, grades, uncertainty features, calibrated decisions, and aggregate tables must be separate immutable layers. Never overwrite raw output with cleaned output.

## 2. Artifact layers

### Layer A — testcase

Source: testcases/r0…r7 JSONL.

Immutable keys:

- testcase_id;
- world_id;
- split;
- dimension and level;
- query;
- ground truth;
- factor profile;
- grader and injection specification.

### Layer B — raw generation

One row per testcase × model × training seed × decoding seed/sample.

Store:

- run_id;
- testcase_id;
- model ID/revision;
- tokenizer revision;
- adapter/checkpoint hash;
- system-prompt hash and exact user prompt (or immutable prompt-artifact URI plus hash);
- decoding parameters;
- raw prompt;
- raw generated text;
- token IDs;
- token log probabilities where available;
- finish reason;
- latency;
- input/output tokens;
- GPU type and seconds;
- timestamp;
- inference engine/version, attention backend, deterministic-kernel mode, tensor/pipeline parallelism, batch schedule, and request-order hash;
- failure/error code.

Raw generation files are append-only and checksummed.

### Layer C — parsed response

One row per greedy/sample response:

- parsed action;
- parsed answer;
- normalized answer;
- parser status;
- abstention flag;
- malformed-response flag;
- extracted intermediate path;
- parser version.

### Layer D — correctness

- factual correctness;
- action correctness;
- commit correctness under the suite-specific operational contract;
- false-commit loss (`commit` and not `commit_correct`);
- stale/current/other label;
- numeric absolute error;
- ambiguity set true/false positives;
- component correctness;
- grader method/version;
- human-review status;
- adjudicated label.

### Layer E — uncertainty

- mean and total negative log-likelihood;
- minimum token probability;
- P(True);
- exact agreement;
- exact answer entropy;
- complexity-profile prediction;
- feature/model revision.

Exact agreement is the sole primary confidence method in protocol v1.2.
P(True) and token values are optional availability-labelled diagnostics. The
removed verbal/semantic/learned/continuous-shift fields are future work.

### Layer F — policy decision

- policy ID;
- contract ID;
- threshold/configuration;
- returned action;
- returned answer/set;
- commit flag;
- commit-correct flag;
- loss value;
- answer coverage;
- set size;
- number of samples used;
- contract supported flag;
- fallback group used.

### Layer G — aggregates

Only generated by versioned scripts:

- per model/dimension/level;
- per uncertainty method;
- per contract/group;
- per hypothesis;
- per seed;
- confidence interval and bootstrap seed.

Confirmatory hypothesis exports must contain exactly one primary result for
each of H1–H5. Secondary endpoints receive distinct `confirmatory=false`
records and cannot enter the Holm family.

## 3. File partitioning

Recommended Parquet layout for scale:

~~~text
results/
  raw_generations/
    model=<id>/dimension=R1/split=test/part-*.parquet
  parsed/
  graded/
  uncertainty/
  decisions/
  aggregates/
  manifests/
  logs/
~~~

CSV templates are supplied for human-readable exports, but use Parquet/Arrow for the million-row generation table.

## 4. Run manifest

Create one manifest before every run:

- code commit or source-tree checksum;
- testcase manifest checksum;
- model and tokenizer revisions;
- environment lock checksum;
- prompt text/hash;
- decoding configuration;
- seed list;
- expected query count;
- expected output count;
- hardware;
- operator;
- start/end time;
- exceptions.

A run is complete only when actual unique keys equal expected keys.

## 5. Generation workflow

1. Filter testcase split and dimension from manifest. For the sealed test, this step is forbidden until the week-20 lock artifacts and preregistration are archived.
2. Resolve model revision locally.
3. Warm up on ten non-study prompts.
4. Run 1% dry batch.
5. Validate response parser and estimate cost.
6. Resume-safe main generation.
7. Verify unique compound key:

\[
(\text{testcase},\text{model},\text{train seed},\text{decode seed})
\]

8. Check no missing IDs.
9. Compute checksum.
10. Mark raw artifact immutable.

Weeks 18–19 execute generation and grading only for fit, tune, and calibrate.
Week 20 consumes those immutable development artifacts to fit, select, and
calibrate the final policy; it does not generate or inspect locked-test output.
Locked-test generation and grading begin for the first time in week 21 and run
once with the frozen pipeline.

## 6. Grading workflow

Order:

1. deterministic normalization;
2. exact/alias/numeric/date grader;
3. action grader;
4. path/set grader;
5. unresolved queue;
6. blinded human annotation;
7. adjudication;
8. frozen final correctness table.

Do not send obvious exact synthetic cases to an LLM judge.

## 7. Human annotation sampling

Use four distinct, non-substitutable audits:

1. **Testcase construction:** the existing 200-row fit/tune sheet, 25
   specifications per R0–R7 suite. This checks query/label/world semantics, not
   model outputs.
2. **Development grader validation:** 300 blinded outputs before the week-20
   lock: 100 R0 outputs for its fixed gate plus 200 stratified R1–R7 outputs
   covering every deterministic grader and action type.
3. **R4 clarification quality:** 200 development-side clarification questions
   and their simulated second-turn outcomes.
4. **Locked-test quality audit:** after the one-shot test run, sample 25 case
   IDs per suite independently of score/correctness (200 total), then join each
   frozen testcase specification to its frozen output. The same 200 rows audit
   both label semantics and output grading. Findings are recorded as deviations
   and cannot change prompts, policies, groups, hypotheses, or thresholds.

In addition, send all genuine automatic-grader conflicts and natural-data
semantic-only matches to blinded review. For each natural dataset, sample

\[
\min\{n,\max(200,\lceil0.05n\rceil)\}
\]

examples for two-annotator quality estimation. Review high-confidence errors
for taxonomy separately; that targeted set must never be reported as an
unbiased error-rate sample. Obvious parser-invalid synthetic outputs retain
their frozen malformed grade and do not require a human to reinterpret them.

Annotators see:

- query;
- reference facts;
- candidate answer;
- rubric.

They do not see:

- model;
- uncertainty method;
- hypothesis;
- whether the answer helps the desired result.

## 8. Quality-control checks

Before scoring:

- record count;
- unique key count;
- missing response rate;
- truncation rate;
- token count distribution;
- prompt hash consistency;
- model revision consistency.

Before analysis:

- split isolation;
- no fit/tune/calibration labels in test-fitting code;
- level balance;
- grader agreement;
- score direction;
- no NaN/inf;
- calibration scores computed only with pre-frozen model;
- threshold chosen without test;
- bootstrap cluster is world/fact.

## 9. Metric computation

Per slice, always report numerator and denominator.

### Factual

- correct/N;
- incorrect/N;
- abstain/N;
- false commit/N;
- incorrect/committed.

### Prediction set

- covered/N;
- set size distribution;
- incorrect candidates per set;
- empty/vacuous outputs.

### Calibration

- Brier;
- log loss;
- calibration intercept/slope;
- ECE with bins declared;
- reliability curve.

### Selection

- risk-coverage curve;
- AURC/E-AURC;
- coverage at fixed risk;
- cost at fixed risk.

## 10. Confidence intervals

- Exact binomial for simple binary cell rates.
- Cluster bootstrap by world for paired/derived metrics.
- Training-seed variability shown separately.
- At least 2,000 bootstrap replicates in final paper.
- Seed bootstrap generator and store replicate summaries.
- Simultaneous intervals for preregistered group guarantees.

## 11. Analysis order

1. R0 gates.
2. Base accuracy curves R1–R6.
3. Uncertainty discrimination.
4. Probability calibration.
5. Global conformal/risk-control validity.
6. Group audit.
7. Deterministic query_rules_v1.1 CCRC utility; no ground-truth oracle profiler
   is required in protocol v1.2.
8. Shift transfer.
9. Required ablations.
10. Hypothesis tests.
11. Optional exploratory interactions, only if separately generated after the core.

This order reduces the temptation to explain away pipeline failures as scientific effects.

## 12. Error taxonomy

Every audited wrong response receives one:

- unknown fact guessed;
- stale value;
- wrong entity;
- precision/rounding;
- ambiguity forced answer;
- missing interpretation;
- invented interpretation;
- component fact unavailable;
- synthesis/composition failure;
- formatting/parser;
- refusal despite known fact;
- contradictory samples;
- consistently wrong/confident;
- other/adjudicated.

Report at least the 200 highest-confidence errors plus a random error sample.

## 13. Reproducible finalization

Before paper freeze:

- regenerate all aggregate files from raw;
- compare hashes;
- verify every table number programmatically;
- generate figures from aggregate artifacts;
- export analysis session information;
- archive the exact test manifest;
- create a read-only release directory.
